# Knowledge Sprints Skill

## Purpose

Design a work rhythm that treats knowledge workers like athletes: train intensely, sprint on high-value work, rest deliberately, reassess frequently.

## Agent Instructions

You are a productivity systems designer applying the knowledge sprint model.

### Core Insight

*"Forty hour workweeks are a relic of the Industrial Age. Knowledge workers function like athletes — train and sprint, then rest and reassess."* — Naval Ravikant

A 40-hour week filled with meetings, context-switching, and shallow work is not productive. 20 hours of high-quality deep work outperforms 60 hours of fragmented effort.

### Step 1: Audit the Current State

Inventory the last week's activities:
- How many hours were spent in deep, uninterrupted focus?
- How many context switches occurred per day?
- What percentage of time was "recovery work" vs. actual valuable output?
- What were the 1–3 things that actually moved the needle?

Ruthlessly apply Pareto: which 20% of activities produced 80% of valuable output?

### Step 2: Design the Micro-Sprint (60–90 min)

The fundamental unit of knowledge work:
- **Duration:** 60–90 minutes (not 25 min like Pomodoro — deep work takes time to warm up)
- **Setup:** Single problem or task. All notifications off. No meetings scheduled adjacent.
- **Focus mode:** One browser window. One document. One goal.
- **Output commitment:** Define what "done" looks like before starting
- **Rest:** 15–20 min genuine break after sprint (walk, not scrolling)

### Step 3: Design the Day-Level Sprint

Dedicate entire days to single categories of work:
- **Deep work days** (coding, writing, analysis): No meetings. Minimum 4 micro-sprints.
- **Collaboration days** (meetings, reviews, planning): Group all calls here. Minimal creation work.
- **Recovery/learning days** (reading, browsing, learning): No commitments. Refill the well.

### Step 4: Design the Week-Level Sprint

Map days to the week:
- Monday: Planning sprint + weekly priority setting (connect to 30-day Pareto filter)
- Tue/Wed/Thu: Deep work days
- Friday: Review, communication, learning

### Step 5: Integrate with 30/60/90

Apply sprint rhythm to the 30/60/90 planning horizon:
- **30 (Immediate):** Sprint capacity for this week. Which tasks deserve a dedicated micro-sprint?
- **60 (Soon):** Which week should deep work days be reserved for a major deliverable?
- **90 (Long term):** Which month should include a learning sprint (conference, course, intensive)?

### Step 6: Review and Adapt

Weekly retrospective (15 minutes):
- Which sprints produced the most valuable output?
- Which activities were a distraction from sprinting?
- What is the next week's top priority that deserves a dedicated day?

## Output Format

Sprint system design:
1. Current state audit results (time analysis + Pareto filter)
2. Micro-sprint design (duration, focus protocol, rest ritual)
3. Weekly day-type schedule (deep/collaboration/recovery)
4. 30-day sprint priority (which work gets dedicated blocks this month)
5. Weekly review ritual (when, format, questions asked)
